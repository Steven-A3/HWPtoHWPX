# Real Tab Definitions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Emit real `<hh:tabProperties>` (7 tabPr, per-tab-stop switch/case/default tabItems) and make paraPr reference its real `tabPrIDRef`, clearing the `tabItem`×212, `switch`×106, `tabPr`×6 header misses.

**Architecture:** Extend the existing 4 layers. New `HwpTab`/`HwpTabDef`/`TabItem`/`TabDef` dataclasses; the reader parses `IdMappings/TabDef` and `ParaShape@tabdef-id` and clamps tab refs; a new `mapper/tab.py` transforms them; `header_writer` emits the real block reusing the paraPr switch idiom.

**Tech Stack:** Python 3.9+, lxml, pyhwp (`hwp5proc xml`), pytest.

## Global Constraints

- **Python 3.9 floor:** NO PEP 604 `X | None` union syntax. Use bare `field: T = None` / `field(default_factory=list)`.
- **Tab transforms (verified):** `Tab@pos` → `case` tabItem pos = raw÷2 (floor toward zero), `default` tabItem pos = raw; `Tab@kind` → `type` (left→LEFT, right→RIGHT, center→CENTER, decimal→DECIMAL, else LEFT); `Tab@fill-type` → `leader` (0→NONE, 1→DOT, 2→DASH, 3→DASH, 4→DASHDOT, else NONE); `TabDef@autotab-left`/`autotab-right` → `autoTabLeft`/`autoTabRight`.
- **tabItem shape:** `case` tabItem carries `unit="HWPUNIT"`; `default` tabItem does NOT. Attribute order on tabItem: pos, type, leader, [unit].
- **Empty TabDef** (no tabs) → self-closing `<hh:tabPr id autoTabLeft autoTabRight/>`.
- **Empty-tab_defs fallback:** if the document has no TabDef, emit the single default `<hh:tabPr id="0" autoTabLeft="0" autoTabRight="0"/>` so `tabPrIDRef="0"` resolves.
- **No dangling:** paraPr `tabPrIDRef` = `ParaShape@tabdef-id` clamped to `[0, tabDefCount-1]` (or 0 if none).
- **Test runner:** `.venv/bin/python -m pytest` — plain `python` lacks `hwp5proc` (~13 spurious failures). Current suite: 125 passing.
- **Switch idiom:** reuse the existing `_hp("switch")`/`_hp("case")`/`_hp("default")` pattern and `case.set("{%s}required-namespace" % NS["hp"], "http://www.hancom.co.kr/hwpml/2016/HwpUnitChar")` already used for paraPr in `header_writer.py`.

```python
def _parse_tab_defs(id_mappings):
    out = []
    for i, el in enumerate(id_mappings.findall("TabDef")):
        tabs = []
        arr = el.find("Array")
        if arr is not None:
            for t in arr.findall("Tab"):
                tabs.append(HwpTab(
                    pos=_int(t.get("pos")),
                    kind=(t.get("kind") or "left").lower(),
                    fill_type=_int(t.get("fill-type")),
                ))
        out.append(HwpTabDef(
            index=i,
            auto_tab_left=_int(el.get("autotab-left")),
            auto_tab_right=_int(el.get("autotab-right")),
            tabs=tabs,
        ))
    return out
```

```python
"""Map HWP tab definitions to OWPML tab definitions."""
from ..owpml.model import TabDef, TabItem

_TYPE = {"left": "LEFT", "right": "RIGHT", "center": "CENTER", "decimal": "DECIMAL"}
_LEADER = {0: "NONE", 1: "DOT", 2: "DASH", 3: "DASH", 4: "DASHDOT"}


def map_tab_defs(tab_defs):
    out = []
    for td in tab_defs:
        items = [TabItem(
            pos=t.pos,
            type=_TYPE.get((t.kind or "left").lower(), "LEFT"),
            leader=_LEADER.get(t.fill_type, "NONE"),
        ) for t in td.tabs]
        out.append(TabDef(
            id=td.index,
            auto_tab_left=td.auto_tab_left,
            auto_tab_right=td.auto_tab_right,
            tabs=items,
        ))
    return out
```
